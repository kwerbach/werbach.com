Had er særligt ved denne vejledning

Mange har udgivet fremragende HTML-vejledninger på WWW - hvorfor så komme med endnu en?

Dette dokument er et opslagsværk, hvor du kan finde det korrekte format på de HTML-koder, du har brug for når du skal skrive dine HTML-sider. De fleste andre HTML-vejledninger er mere af en beskrivende karakter. Hvis du er nybegynder, skal du måske prøve en af de andre vejledninger først.

Der findes også flere anbefalinger til "god stil"; eg vil absolut anbefale dig at læse en eller flere af disse, da det er et område der overses af alt for mange Web-forfattere.

Min WWW hjælpeside har referencer til mange steder, der indeholder HTML beskrivelser. Det er et godt sted at starte, hvis du har spørgsmål der går ud over denne beskrivelse, der bevidst er "skåret ind til benet".

Denne vejledning er for folk, der ikke behøver skridt-for-skridt vejledninger, men som ikke kan alle HTML-koder udenad. Hvis du vil have en "quick reference" som giver dig det nødvendigste på ét sted, befinder du dig det rette sted. På de reaktioner jeg har fået ser det ud til, at det omfatter temmelig mange mennesker.

Hvilke HTML-koder er medtaget

En vigtig egenskab ved denne vejledning er, hvad den indeholder. De fleste andre HTML-vejledninger indeholder få eller ingen af Netscapes udvidelser til HTML. De udelukker også ofte HTML 3.0 koder, fx tabel-definitioner.

Denne vejledning skærer ind til benet tekstm ssigt; men indholdsm ssigt medtager den alle koder, der er i almindelig brug i dag.. Specifikt indeholder den følgende:

  • HTML 2.0 koder
  • Nogle foreslåede HTML 3.0 koder
  • Netscapes udvidelser til HTML

HTML 2.0 er den nyeste almindeligt udbredte standard, så HTML 2.0 koder burde fungere i alle nyere Web browsere. Se http://www.w3.org/ for mere information. Medtaget her er alle koder fra 15. juni 1995 memoet fra "the HTML Working Group", med undtagelse af forældede koder som fx <plaintext>, og visse koder, som ikke er i almindelig brug, eller som kræver avancerede server scripts.

Jeg har medtaget alle Netscape tags til og med version 1.1N, undtagen de som dynamisk sammekæder dokumenter. Netscapes udvidelser er ikke en del af HTML specificationen, og vil muligvis ikke blive vist korrekt af alle browsere. Netscape Navigator har imidlertid, iflg de nyeste undersøgelser, over 70% af browser markedet, og det er derfor jeg har medtaget koderne her; om du vil bruge dem eller ej er op til dig selv.

Endelig har jeg også medtaget nogle HTML 3.0 koder, der understøttes af de førende browsere (fx Netscape Navigator, Microsoft Explorer og NCSA Mosaic, som tilsammen dækker over 90% af markedet). HTML 3 er pt. under udarbejdelse af "the HTML Working Group", og version 3.2 forventes fastlåst i 1996. Hvordan "standardkrigen" kommer til at ende, står hen i det uvisse.

Formattering af dette dokument

For tydelighedens skyld har jeg skrevet forskellige parametre til den samme HTML-kode på hver sin linie. I almindelighed kan parametrene kombineres.

Anvendte symboler

URL    URL adresse på en ekstern fil
?      Vilkårligt tal (dvs <H?> betyder <H1>, <H2>, <H3>, osv.)
%      Vilkårlig procent (dvs <HR WIDTH=%> betyder <HR WIDTH=50%>, osv.)
***    Vilkårlig tekst (dvs ALT="***" betyder udfyld med tekst)    
$$$$$$ Vilkårlig hex (i.e. BGCOLOR="#$$$$$$" betyder BGCOLOR="#00FF1C", osv.)
,,,    Komma-separeret (dvs COORDS=",,," betyder COORDS="0,0,50,50", osv.)
|      Alternativer (dvs ALIGN=LEFT|RIGHT|CENTER betyder vælg en af disse)

Kompatibilitet

(husk - HTML udvikler sig, og browsere opfører sig forskelligt)
       (ingen angivelse) I HTML 2.0 spec.; bør virke med alle browsere 
N1.0   Netscape udvidelse introduceret med Navigator version 1.0
N1.1   Netscape udvidelse introduceret med Navigator version 1.1
N2.0   Netscape udvidelse introduceret med Navigator version 2.0
3.0    HTML 3.0 kode implementeret af Netscape eller andre
+      Denne Netscape udvidelse understøttes af de fleste browsere

Version History

 Historikken citeres fra den engelsksprogede udgave, da ingen tidligere udgaver fandtes på dansk.

Version 2.0 -- Lots of changes and corrections in this version

  • Rewrote the introductory material and compatibility table language
  • Added + notation for Netscape extensions that have been adopted by most major browsers. This notation is by nature approximate; it represents my judgment about which tags have are widely supported even though they are not (and may never be) in HTML 3.0.
  • Added ,,, notiation in table of symbols for comma-delimited lists.
  • Restructured the Hard Formatting and Soft Formatting sections into Presentation Formatting and Structural Definition. This change makes clear the important distinction between presentation-based and structural markup. A few tags have been moved around as a result of this change, and the <ADDRESS> tag has been moved up from Miscellaneous into Structural Definition.
  • Eliminated the Alignment section by shifting the tags listed there into either Structural Definition or Presentation Formatting. Although this results in the Netscape <CENTER> tag being split up from the other alignment tags, it allows the alignment attributes of tags such as <P> and <H?> to be associated with the underlying tags they operate on. This change is consistent with the HTML 3.0 philosophy of handling alignment through attributes to existing tags rather than through dedicated tags.
  • Added listing for Netscape client pull animation (this has been around since Netscape 1.1, but was not listed in the Guide).
  • Support for new Netscape and HTML 3.0 tags implemented in Netscape 2.0:
    • Frames (added a whole new section)
    • Embedded Objects (in Links and Graphics)
    • Divisions (in Structural Definition)
    • Font Color (in Presentation Formatting)
    • Big/Small font size (in Structural Definition)
    • Superscript/Subscript (in Presentation Formatting)
    • File Upload (in Forms)
    • Text wrap (in Forms)
    • Target Windows (in Links and Graphics) and Base Target (in Miscellaneous)
    • Client-side imagemaps (in Links and Graphics)
  • Removed <XMP> tag, because I have been convinced that it is archaic.
  • Moved prologue tags to Miscellaneous section, to indicate that they are not required and reduce confusion.
  • Fixed listing for <NOBR> container by adding a closing tag
  • Changed "outline type" to "numbering type" for Netscape extension to ordered lists, in order to avoid confusion about what this attribute does.
  • Changed wording of <P> tag description to indicate that it is "usually" rendered as a double return.
  • Changed notation for hex triplet from $$$ to $$$$$$ to indicate that six characters are required to reference the three hex values.
  • improved consistancy of column alignment in the text version.

Version 1.21 -- July 26, 1995. Removed the archaic <NEXTID> tag and added some annotations to the HTML and table versions.

Version 1.2 -- July 25, 1995. Several revisions to improve compliance with the official HTML specifications, and to acknowledge more widespread implementation of certain HTML 3.0 tags.

Version 1.1 -- July 19, 1995. Cleaned up a number of things in the introductory material, and correctly specified the ISO 8859-1 character set instead of ASCII.

Version 1.03 -- July 12, 1995. Updated copyright notices to eliminate confusion with Bare Bones Software. Added Netscape relative font sizing tag, and changed the link to the list of ASCII symbol codes.

Version 1.02 -- April 24, 1995. Corrected some additional typos, fixed the link to this version history, and added the standard copyright and registered TM codes to go along with the Netscape versions.

Version 1.01 -- April 11, 1995. Fixed a couple minor typos and ambiguities pointed out by helpful folks, including some errors in the list tags section.

Version 1.0 HTML -- April 6, 1995. Has some additional introductory material, but the text is otherwise identical to version 1.0. Formatted for WWW viewing, with assistance from John Berkin.

Version 1.0 -- April 2, 1995. Corrected some errors pointed out by other people, and made some minor revisions to the introduction and the formatting of certain sections.

Version 0.9b -- March 30, 1995. First public version. Posted on comp.infosystems.www.providers.
