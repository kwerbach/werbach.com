本導引的特點

很多人已經作了許多很好的HTML導引手冊放在網路上，誰還需要其他的導引？

這份導引的設計目的在於補足那些我所讀過的導引中疏漏的部份。我希望它簡單扼要，可以在你建立HMTL文件時當作"「小抄」"使用。大部份的HTML導引著重於提供詳盡的描述，以及HTML的一般特性。 如果你是初學者」最好先看看那一類的導引。

還有一些有關 HTML風格設計的導引，著重於如何寫出好看又適用於不同的瀏覽器的頁面。如果你正打算從事HTML頁面的創作的話，我誠心的鼓勵你起碼找一份那樣的導引來讀一讀。確實有太多的人忽略了這一個事實：他們的頁面是要給別人看的。方法總有好壞，說不定換一換作法也許會更好。

我的 WWW頁面中的求助頁裡有一些連結指到提供建立HTML頁面相關資訊的WWW資訊站，如果你不滿足於本導引所提供的資訊，也許那裡是個好的起點。

本導引為熟悉HTML的人而設計，他們已經不需要一步一步地引導，只不過他們還沒有把正確的使用格式背起來。 如果你需要一個"速查表"，用來查照，而不需要閱讀額外的描述的話，本導引就是你要的。以我所收到的反應看來，以前有這樣的參考表的人似乎不多。

收錄標籤的範圍

本導引的另一項特色是它所收錄的標籤。大部份的導引只包含很少的或根本沒有收錄Netscape所擴充的標籤，甚至也沒有把HTML 3.0的標籤收錄進來。例如，「表格」已經可以在Netscape以及Mosaic上看到了，有些導引卻還沒有收錄。

另外，本導引也企圖要收錄當今在所有頁面上使用的標籤。特別是下列主題

  • HTML 3.2 標籤
  • Netscape 的擴充標籤

超文標記語言的發展過程一言難盡，制訂標準的腳步往往趕不上市場的實際發展。World Wide Web Consortium是制訂標準的正式組織，一九九六年五月的時候，它發佈了超文標記語言3.2版的草案，該草案設計初衷是為了整合並標準化當時廣為接受的實際應用中的標記。其中， W3C建議編寫抄文標記語言文件的人使用3.2版的標記，因此，我把這些標記收錄進來了，不論他們原來屬於哪一版的，或是Netscape附加的，這些標記應該能用於大部分的瀏覽器。

我也增加了 Netscape （網景）Navigator 3.0b5版附加的標記，除了部分標記，像是「伺服端推動」的動畫以外，其它的標記在其它的瀏覽器中顯示會有問題。所以在使用這類標記之前，最好考慮一下您的觀眾的情況。

本文件的格式

為了簡明起見，我把可以用於同一標籤的不同屬性分開成不同的列，一般而言，你可以把不同的屬性寫在一起。

記號說明

URL    其他檔案的URL（如果在同一目錄內的話，只要檔名就可以）
?      任意數字 （例如： <H?>意思是<H1>, <H2>,<H3>,等等）
%      任意比率（例如： <HR WIDTH=%>意思是 <HR WIDTH=50%>, 等等）
***    任意文字 （例如 ALT="***" 意思是填入文字）
$$$$$$ 任意十六進位（例如： BGCOLOR="#$$$$$$" 意思是BGCOLOR="#00FF1C"等等）
,,,    半形逗點分開的欄位（例如： COORDS=",,,"意思是COORDS="0,0,50,50"等等）
|      選擇其一（例如：ALIGN=LEFT|RIGHT|CENTER 意思是選擇其中一個）

適用範圍
（請記住，HTML處與發展中，而且瀏覽器的接受能力不同）

       （沒有標記）表示在HTML 2.0 規格內，所有瀏覽器都應適用。
N1.0   Netscape 擴充，首見於1.0版
N1.1   Netscape 擴充，首見於1.1版
N2.0   Netscape 擴充，首見於2.0版
3.0    HTML 3.0 標籤 可適用於 Netscape 或其他主要的瀏覽器
+      本是 Netscape 擴充的標籤，不過其他主要的瀏覽器也可適用。

版本史

Version 3.0 -- July 21, 1996.

  • 使本導引與 HTML 3.2 規格相符合
  • 新增加Java相關的標記
  • 修訂說明與註解
  • 修訂「表格版」的版面
  • 因為大部分瀏覽器都能支援表格，所以把其它版面的版本都省略了。無法看見表格版的人，仍可以選用純文字版。
  • 新增 Netscape Navigator 3.0及 3.0b5附加的標記。包括：
    • 表格儲存格底色
    • 多欄位
    • 留白
    • 視框邊線及其控制
    • 引用<S> 標記來表示刪除，並引用 <U> 標記來表示底線.。 這些在超文標記語言3.0中有，不過卻不在3.2版中，所以我把它們列為Netscape的附加標記。
    • 增加 <FONT> 標記的指定字形功能 (原本為Microsoft的附加標記)
  • 以及其他的打字錯誤

Version 2.0 --在本版中有很多更改及校正

  • 改寫介紹文件，以及可適用於表格的語言
  • 增加「+」這個記號。這個記號是我個人的判斷，它表示了哪一個標籤已經廣為適用，不過他們可能不一定會成為HTML 3.0的規格。
  • 增加「 ,,, 」這個記號。
  • 調整「硬格式」（ Hard Formatting） 與「軟格式」（ Soft Formatting ）這兩節， 成為「與外觀相關的標籤」（Presentation Formatting）與「結構性定義」（Structural Definition）兩節。這一改變使得「表現性標記」（presentation-based markup）與「結構 性標記」（structural markup）標籤間的差別更明顯。有些標籤因此被搬移，例如，< ADDRESS>這個標籤從「雜項」（Miscellaneous ）中被搬到「結構性定義」這一節。
  • 刪除「看齊」（Alignment） 這一節，將其中的標籤打散到「結構定義」與「與外觀相 關的標籤」這兩節中。雖然，這使得Netscape 的<CENTER>標籤也被從「看齊」這節 中踢出來，不過這也使得具有排列性質的標籤，例如，<P>以及 <H?>可以和那 些被它們作用的標籤放在一起。這一改變與HTML 3.0 的觀念相符，也就是說，以「屬性」 的方式看待排列的問題，而不是一昧新增標籤。
  • 增加 有關Netscape「 客戶端拉」（client pull）動畫功能的項目（Netscape 1.1 版以後就有了，不過以前本導引沒有加入）
  • 支援適用於Netscape 2.0以及HTML 3.0的標籤。
    • 視框（FRAME）（新增全新的一節）
    • 內嵌物件（在「連結與圖形」）
    • 分隔（Divisions）（在「結構定義」）
    • 字體顏色（在「與外觀相關的標籤」）
    • 放大／縮小字體大小（在「與外觀相關的標籤」）
    • 上標／下標（在「與外觀相關的標籤」）
    • 檔案上載（在「表單」）
    • 文字換行（在「表單」）
    • 目標視窗（在「連結與圖形」）以及基本目標視窗（在「雜項」）
    • 客戶端點選圖（Client-side imagemaps）（在「連結與圖形」）
  • 刪除 <XMP> ，因為我相信它已經過時了。
  • 把檔案型態聲明的標籤搬到「雜項」，以表示它們不是必要的，也減少一些可能的混淆。
  • 增加一個結束標籤以修改有關 <NOBR> 的列舉。
  • 為了避免混淆不清，我把Netscape擴充的有序性列舉"outline type"改成 "numbering type"。
  • 修改 <P> 的解說，以指出本標籤"通常"意味著兩個「 return」 （空一行）。
  • 把三個十六進位數字的記號從$$$變成 $$$$$$ 以表示用六個數字以表示這三個十六 進位數。
  • 增進純文字版本中欄位對齊的一致性。

使用中文版的相關問題

  • 字型選擇：因為有些字形的空格比較小，請選用細明體，或是其他可以「二個空格大小等於一個中文字」的字型。
  • 使用Mosaic的人，請更改preformatted font成12點字的「細明體」。
  • 我在每一份文件的文頭區段內增加了 <META HTTP-EQUIV="Content-Type" CONTENT="text/html; charset=big5" > 這一段標記，以使得Netscape瀏覽器自動切換解碼方式。 如果文字超過您的螢幕範圍，建議您把設定字型變小。至於使用國標 gb碼的人， 因為我不知道該如何設定 charset的希望有人能 告訴我
