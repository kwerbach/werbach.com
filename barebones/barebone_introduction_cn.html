本导引的特点

很多人已经作了许多很好的HTML导引手册放在网路上，谁还需要其他的导引？

这份导引的设计目的在於补足那些我所读过的导引中疏漏的部份。我希望它简单扼要，可以在你建立HMTL文件时当作"「小抄」"使用。大部份的HTML导引著重於提供详尽的描述，以及HTML的一般特性。 如果你是初学者」最好先看看那一类的导引。

还有一些有关 HTML风格设计的导引，著重於如何写出好看又适用於不同的浏览器的页面。如果你正打算从事HTML页面的创作的话，我诚心的鼓励你起码找一份那样的导引来读一读。确实有太多的人忽略了这一个事实：他们的页面是要给别人看的。方法总有好坏，说不定换一换作法也许会更好。

我的 WWW页面中的求助页里有一些连结指到提供建立HTML页面相关资讯的WWW资讯站，如果你不满足於本导引所提供的资讯，也许那里是个好的起点。

本导引为熟悉HTML的人而设计，他们已经不需要一步一步地引导，只不过他们还没有把正确的使用格式背起来。 如果你需要一个"速查表"，用来查照，而不需要阅读额外的描述的话，本导引就是你要的。以我所收到的反应看来，以前有这样的参考表的人似乎不多。

收录标签的□围

本导引的另一项特色是它所收录的标签。大部份的导引只包含很少的或根本没有收录Netscape所扩充的标签，甚至也没有把HTML 3.0的标签收录进来。例如，「表格」已经可以在Netscape以及Mosaic上看到了，有些导引却还没有收录。

另外，本导引也企图要收录当今在所有页面上使用的标签。特别是下列主题

  • HTML 3.2 标签
  • Netscape 的扩充标签

超文标记语言的发展过程一言难尽，制订标准的脚步往往赶不上市场的实际发展。World Wide Web Consortium是制订标准的正式组织，一九九六年五月的时候，它发布了超文标记语言3.2版的草案，该草案设计初衷是为了整合并标准化当时广为接受的实际应用中的标记。其中， W3C建议编写抄文标记语言文件的人使用3.2版的标记，因此，我把这些标记收录进来了，不论他们原来属於哪一版的，或是Netscape附加的，这些标记应该能用於大部分的浏览器。

我也增加了 Netscape （网景）Navigator 3.0b5版附加的标记，除了部分标记，像是「伺服端推动」的动画以外，其它的标记在其它的浏览器中显示会有问题。所以在使用这类标记之前，最好考虑一下您的观众的情况。

本文件的格式

为了简明起见，我把可以用於同一标签的不同属性分开成不同的列，一般而言，你可以把不同的属性写在一起。

记号说明

URL    其他档案的URL（如果在同一目录内的话，只要档名就可以）
?      任意数字 （例如： <H?>意思是<H1>, <H2>,<H3>,等等）
%      任意比率（例如： <HR WIDTH=%>意思是 <HR WIDTH=50%>, 等等）
***    任意文字 （例如 ALT="***" 意思是填入文字）
$$$$$$ 任意十六进位（例如： BGCOLOR="#$$$$$$" 意思是BGCOLOR="#00FF1C"等等）
,,,    半形逗点分开的栏位（例如： COORDS=",,,"意思是COORDS="0,0,50,50"等等）
|      选择其一（例如：ALIGN=LEFT|RIGHT|CENTER 意思是选择其中一个）

适用□围
（请记住，HTML处与发展中，而且浏览器的接受能力不同）

       （没有标记）表示在HTML 2.0 规格内，所有浏览器都应适用。
N1.0   Netscape 扩充，首见於1.0版
N1.1   Netscape 扩充，首见於1.1版
N2.0   Netscape 扩充，首见於2.0版
3.0    HTML 3.0 标签 可适用於 Netscape 或其他主要的浏览器
+      本是 Netscape 扩充的标签，不过其他主要的浏览器也可适用。

版本史

Version 3.0 -- July 21, 1996.

  • 使本导引与 HTML 3.2 规格相符合
  • 新增加Java相关的标记
  • 修订说明与注解
  • 修订「表格版」的版面
  • 因为大部分浏览器都能支援表格，所以把其它版面的版本都省略了。无法看见表格版的人，仍可以选用纯文字版。
  • 新增 Netscape Navigator 3.0及 3.0b5附加的标记。包括：
    • 表格储存格底色
    • 多栏位
    • 留白
    • 视框边线及其控制
    • 引用<S> 标记来表示删除，并引用 <U> 标记来表示底线.。 这些在超文标记语言3.0中有，不过却不在3.2版中，所以我把它们列为Netscape的附加标记。
    • 增加 <FONT> 标记的指定字形功能 (原本为Microsoft的附加标记)
  • 以及其他的打字错误

Version 2.0 --在本版中有很多更改及校正

  • 改写介绍文件，以及可适用於表格的语言
  • 增加「+」这个记号。这个记号是我个人的判断，它表示了哪一个标签已经广为适用，不过他们可能不一定会成为HTML 3.0的规格。
  • 增加「 ,,, 」这个记号。
  • 调整「硬格式」（ Hard Formatting） 与「软格式」（ Soft Formatting ）这两节， 成为「与外观相关的标签」（Presentation Formatting）与「结构性定义」（Structural Definition）两节。这一改变使得「表现性标记」（presentation-based markup）与「结构 性标记」（structural markup）标签间的差别更明显。有些标签因此被搬移，例如，< ADDRESS>这个标签从「杂项」（Miscellaneous ）中被搬到「结构性定义」这一节。
  • 删除「看齐」（Alignment） 这一节，将其中的标签打散到「结构定义」与「与外观相 关的标签」这两节中。虽然，这使得Netscape 的<CENTER>标签也被从「看齐」这节 中踢出来，不过这也使得具有排列性质的标签，例如，<P>以及 <H?>可以和那 些被它们作用的标签放在一起。这一改变与HTML 3.0 的观念相符，也就是说，以「属性」 的方式看待排列的问题，而不是一昧新增标签。
  • 增加 有关Netscape「 客户端拉」（client pull）动画功能的项目（Netscape 1.1 版以後就有了，不过以前本导引没有加入）
  • 支援适用於Netscape 2.0以及HTML 3.0的标签。
    • 视框（FRAME）（新增全新的一节）
    • 内嵌物件（在「连结与图形」）
    • 分隔（Divisions）（在「结构定义」）
    • 字体颜色（在「与外观相关的标签」）
    • 放大／缩小字体大小（在「与外观相关的标签」）
    • 上标／下标（在「与外观相关的标签」）
    • 档案上载（在「表单」）
    • 文字换行（在「表单」）
    • 目标视窗（在「连结与图形」）以及基本目标视窗（在「杂项」）
    • 客户端点选图（Client-side imagemaps）（在「连结与图形」）
  • 删除 <XMP> ，因为我相信它已经过时了。
  • 把档案型态声明的标签搬到「杂项」，以表示它们不是必要的，也减少一些可能的混淆。
  • 增加一个结束标签以修改有关 <NOBR> 的列举。
  • 为了避免混淆不清，我把Netscape扩充的有序性列举"outline type"改成 "numbering type"。
  • 修改 <P> 的解说，以指出本标签"通常"意味著两个「 return」 （空一行）。
  • 把三个十六进位数字的记号从$$$变成 $$$$$$ 以表示用六个数字以表示这三个十六 进位数。
  • 增进纯文字版本中栏位对齐的一致性。

使用中文版的相关问题

  • 字型选择：因为有些字形的空格比较小，请选用细明体，或是其他可以「二个空格大小等於一个中文字」的字型。
  • 使用Mosaic的人，请更改preformatted font成12点字的「细明体」。
  • 我在每一份文件的文头区段内增加了 <META HTTP-EQUIV="Content-Type" CONTENT="text/html; charset=big5" > 这一段标记，以使得Netscape浏览器自动切换解码方式。 如果文字超过您的萤幕□围，建议您把设定字型变小。至於使用国标 gb码的人， 因为我不知道该如何设定 charset的值，希望有人能 告诉我
