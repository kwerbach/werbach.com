このガイドの独自な点

何人もの人が優れた HTML ガイドを作り Web ページに発表している -- この上別のものを誰が必要とするだろうか？

この文書は特殊な役割を担うために考案された。Web ページ作成時に正しいタグ形式を参照するために使う手軽な "カンニングペーパー" になればと思う。他の HTML ガイドは、特定のタグの使用法や HTML の一般的な働きについてもっと多くの説明資料を提供することを目的としている。あなたが初心者なら、そういったガイドの一つを最初にトライしてみた方がいいかもしれない。

様々なブラウザに対応し、見栄えのするページの書き方についてもっと焦点を絞ったスタイルガイドもいくつか存在する。もしあなたが Web ページを作成しようと考えているならそのどれかを読むことをお薦めする。自分のページは他人に読まれるもので、そのやり方には良いものも悪いものもあるのだという事実を無視する人々があまりにも多いからだ。

私の WWW Help Page では HTML と Web ページデザインに関する情報を提供している多数の他のサイトにリンクを張ってある。もしあなたがここにある " Bare Bones（骨子）な" 情報を越える特別な疑問があるなら、ここは出発点とするにはいい場所である。

このガイドは、段階を追ったインストラクションの必要がない程度に知識はあるが、各 HTML タグの正確な形式をまだ憶えていない人々を対象に考案されている。 あなたが付随的な説明資料を読まなくてもちょっと見られる "早見表" が欲しいなら、このガイドはあなたのためにある。

どんなタグが含まれるか

このガイドのもう一つの特異な点はこれが含むものである。The Bare Bones Guide to HTML は今日 Web ページで通常使用されるタグをすべて含めようとしている。 f特にこのガイドでは以下のものがリストされている：

  • HTML 4.0 タグ
  • Netscape と Microsoft 拡張 HTML

HTML の開発は込み入ったプロセスだった。それは公式の標準仕様を採用しようとする試みは業界のデベロッパーによって打ち砕かれるのが常だったからだ。World Wide Web Consortium（W3C）は HTML のための公式標準化団体であり、現行の W3C 勧告は HTML 4.0 である。

その Position Statement on HTML の中で、W3C は情報提供者が HTML 4.0 仕様を使い、一方ツールも旧バージョンとの互換性を保つように勧告している。したがって、バージョン 4.0 の Bare Bones Guide to HTML にはすべての HTML 4.0 タグを含めたが、同時に、バージョン 4.0 で導入されたタグはバージョン 3.2 のものと区別している。

また、サーバープッシュ・アニメーションのようなサーバー側インタラクションタグのいくつかの例外を除いて、すべての Netscape と Microsoft 拡張 HTML も含めた。これらの拡張タグの中には他のブラウザであなたのページを見た場合に正しく表示しないものもあるので、使うかどうか決める前にあなたの読者のことを考えてみるべきだ。

この文書のフォーマット

明瞭にするために、同じタグに適用できる異なる属性を別の行に分割した。一般的に、複数の属性が同じタグ内で結合可能だ。

読みやすくするためにタグは大文字でリストしたが、ほとんどのタグは大文字小文字を認識しない。

使用した記号

URL    外部ファイルの URL （または同ディレクトリ内ならファイル名だけ）
?      任意の数字（たとえば <H?> は <H1>、 <H2>、 <H3> 等の意味）
%      任意のパーセンテージ（たとえば <HR WIDTH="%"> は <HR WIDTH="50%"> 等の意味）
***    任意のテキスト（たとえば ALT="***" はテキストをはめ込むという意味）
$$$$$$ 任意の 16 進数（たとえば BGCOLOR="#$$$$$$" は BGCOLOR="#00FF1C" 等を意味）
:::    任意の日付（たとえば DATETIME=":::" は "1994-11-05T08:15:30" 等の意味）
@      電子メールアドレス（たとえば "mailto:@" は "mailto:kevin@werbach.com" 等の意味）
,,,    コンマ区切（たとえば COORDS=",,," は COORDS="0,0,50,50" 等を意味）
|      選択肢（たとえば ALIGN=LEFT|RIGHT|CENTER は選択肢の一つを選ぶという意味

バージョン履歴

バージョン 4.0 -- 1999 年 1 月

  • HTML 4.0 仕様に従ってすべてを更新
  • メインのインデックスページのレイアウトを再デザイン。
  • すべての主要ブラウザが現在テーブルに対応したので、テーブル版とフォーマット版を統合

バージョン 3.0 -- 1996 年 7 月 21 日

  • HTML 3.2 仕様に適合させるためにすべてを更新
  • HTML 3.2 でサポートされたJava たぐのための新しい節を追加
  • イントロダクションと注釈を改訂
  • 読みやすくするために“テーブル”版のレイアウトを再デザイン
  • すべての主要ブラウザはテーブルに対応するようになったので、時間を節約するためにこの Guide の別にあった非テーブル“HTML”版をやめ、“テーブル”版を“フォーマット”版と名前を変えることにした。このバージョンを表示できないユーザーは引き続き“プレーンテキスト”版を使うことができる。
  • 3.0b5 までのNetscape Navigator 3.0 のベータ版で実装された次の新しいタグを追加：
    • テーブルセル背景色
    • 複数段組みテキスト
    • スペーサー
    • フレーム枠のコントロール
    • <S> タグを使う削除線と the <U> タグを使う下線。これらは HTML 3.0 にあったが HTML 3.2 にはないので、Netscape 拡張としてリストしてある。
    • <FONT> タグの FACE 属性（初出は Microsoft 拡張）
  • 誤字を修正

バージョン 2.0 -- 1995 年 10 月 7 日 このバージョンでの多くの変更点と修正点

  • 表組み表記に新しく導入されたものとコンパチビリティに対応
  • ほとんどの主要なブラウザに対応されている Netscape 拡張のための + 注記を追加この注は基本的にはだいたい正しいもので、HTML 3.0 には（恐らくこの先にも）なくても広くサポートされていると私が判断を下したタグである。
  • コンマ区切のリスト用の記号の欄に ,,, 注記を追加。
  • 「ハードフォーマット」と「ソフトフォーマット」節を「プレゼンテーションフォーマット」と「構造型定義」として編成し直した。この変更はプレゼンテーションのためのマークアップと構造のためのマークアップの重要な区別を明確にするためだ。この変更の結果、いくつかのタグは移動され、 <ADDRESS> タグが「その他」から「構造型定義」に移動してくることになった。
  • 「行揃え」節でリストされていたタグが「構造型定義」または「プレゼンテーションフォーマット」のどちらかに移動したことで、この節は削除された。これで Netscape の <CENTER> タグは他の行揃えタグと分割されることになったが、 <P;> や <H?> のようなタグの行揃え属性が、それらが制御する基礎的なタグと連合できるようになる。この変更は、専用タグよりも現行タグの属性を使って行揃えを扱おうという HTML 3.0 の哲学と矛盾するものではない。
  • Netscape クライアントプルアニメーションのためのリストを追加（これは Netscape 1.1 のころからあるが、このガイドではリストされていなかった）。
  • Netscape 2.0 で実装された新 Netscape タグと HTML 3.0 タグをサポート：
    • フレーム（まったく新しい節を追加）
    • 埋め込みオブジェクト（リンク節とグラフィック節）
    • 仕切り（構造型定義節）
    • フォント色（プレゼンテーションフォーマット節）
    • 大／小フォントサイズ（構造型定義節）
    • 上つき文字／下つき文字（プレゼンテーションフォーマット節）
    • ファイルアップロード（フォーム節）
    • テキスト折り返し（フォーム節）
    • 目標ウインドウ（リンク節とグラフィック節）とベース目標（その他節）
    • クライアント側イメージマップ（リンク節とグラフィック節）
  • 以前から遺物だと思っていたので <XMP> タグを削除。
  • 序タグをその他節に移動した。これは必須ではないことを示し、混乱を少なくするためである。
  • 閉タグを追加することで <NOBR> コンテナのリストを修正。
  • 番号なしリスト Netscape 拡張で "アウトライン型" を "番号型" に変更。これはこの属性がなんのためのものか混乱を避けるためである。
  • <P> タグの説明文を、"通常"は２回の改行で表現されるということを示すものに変更。
  • ３組の 16 進数注記を $$$ から $$$$$$ に変更。これは３つの 16 進数値を参照するためには６文字が必要とされるということを指し示すためである。
  • テキスト版で、コラム内の行揃えの一貫性を改良。

バージョン 1.22 -- 1995 年 8 月 18 日　フォーム節からたまたま省かれていたいくつかのタグと属性を追加し、どのカラータグが HTML 3.0 仕様にリストされているかを修正。

バージョン 1.21 -- 1995 年 7 月 26 日　古めかしいタグと <NEXTID> タグを取り除き HTML と表組みバージョンへの注釈をいくつか追加。

バージョン 1.2 -- 1995 年 7 月 25 日　公式な HTML 仕様書へより準拠させるためと特定の HTML 3.0 タグの実装がこれほど広く行き渡っていることを認めるためのいくつかの改訂。

バージョン 1.1 -- 1995 年 7 月 19 日　マテリアルの紹介内の多くを書き換え、 ASCII の替わりに ISO 8859-1 文字セットを正確に指定。

バージョン 1.03 -- 1995 年 7 月 12 日　Bare Bones Software との混乱を取り除くためにコピーライト記述を更新。 Netscape 関連のフォントサイズタグを追加し、 ASCII 記号コードのリストへのリンクを変更。

バージョン 1.02 -- 1995 年 4 月 24 日　追加された誤植をいくつか修正、このバージョン履歴へのリンクを修正、 Netscape バージョンに付随する標準のコピーライトと登録 TM コードを追加。

バージョン 1.01 -- 1995 年 4 月 11 日. 親切な人達から指摘された２つの小さな誤植とあいまいな表現を修正、リストタグ節にあるエラーのいくつかを含む。

バージョン 1.0 HTML -- 1995 年 4 月 6 日. マテリアルの紹介にいくつか追加したが、その他のテキストはバージョン 1.0 と全く同じ。John Berkin 氏の助けを借りて、WWW 表示のためのフォーマット。

バージョン 1.0 -- 1995 年 4 月 2 日 人々から指摘されたいくつかのエラーを修正し、イントロダクションと特定の節のフォーマットへいくつか小さな改訂を施した。

バージョン 0.9b -- 1995 年 3 月 30 日 最初の公開バージョン。comp.infosystems.www.providers にポスト。
